# Interaction Logging System: Redesign for Clarity

## The Problem with Current System

**Current Flow:**
1. Select Mode: "One-on-One" / "Group Flow" / "Celebration" / etc. (6 options)
2. Select Activity: "Coffee" / "Meal" / "Walk" / etc. (6 options per mode = 36 total)
3. Add details

**Issues:**
- ü§î **Cognitive overhead**: "Is coffee one-on-one or cozy-time?"
- üîÑ **Redundancy**: Coffee, Tea Time, Drink all essentially = "had drinks together"
- üì± **Too many taps**: 2 screens before getting to the substance
- üéØ **Mode confusion**: "Quick Touch" mode contains activities like "Quick Visit" - is that the mode or the activity?

---

## New System: Universal Interaction Types + Contextual Tags

### Core Principle
**Log the WHAT quickly, capture the WHY through reflection**

Instead of:
- Mode (how) ‚Üí Activity (what) ‚Üí Details (when/notes)

We have:
- **Interaction Type (what)** ‚Üí **Contextual Reflection (how/why)** ‚Üí Details (when/vibe)

## Scoring System Updates

### Base Scores by Type:

```typescript
const BASE_SCORES = {
  'text-call': 10,
  'voice-note': 12,
  'meal-drink': 22,
  'hangout': 20,
  'deep-talk': 28,
  'event-party': 27,
  'activity-hobby': 25,
  'celebration': 32,
  'acts-of-care': 30,
};
```

### Tag Multipliers:

```typescript
const TAG_MULTIPLIERS = {
  'deep-talk': 1.3,      // Significantly boosts score
  'vulnerable': 1.25,     // Emotional depth matters
  'acts-of-care': 1.3,    // Support is highly valued
  'one-on-one': 1.1,      // Intimate time bonus
  'group-hangout': 0.9,   // Dilution factor
  'first-time': 1.2,      // Novelty bonus (Fool loves this)
  'spontaneous': 1.15,    // Unplanned connection
  'long-overdue': 1.1,    // Reconnection bonus
};
```

### Archetype Adjustments:

```typescript
const ARCHETYPE_TYPE_MULTIPLIERS = {
  HighPriestess: {
    'deep-talk': 1.5,      // Loves meaningful conversation
    'meal-drink': 1.2,     // Values intimate meals
    'event-party': 0.6,    // Draining
  },
  Sun: {
    'celebration': 1.6,    // Thrives on milestones
    'event-party': 1.5,    // High energy events
    'deep-talk': 0.9,      // Less focus on depth
  },
  Hermit: {
    'deep-talk': 1.4,
    'hangout': 1.3,        // Values quiet time
    'event-party': 0.5,    // Very draining
  },
  // ... etc
};
```

### Final Score Calculation:

```typescript
function calculateWeaveScore(
  type: InteractionType,
  vibe: Vibe,
  tags: string[],
  archetype: Archetype,
  duration?: Duration
): number {
  let score = BASE_SCORES[type];

  // Vibe multiplier
  score *= VIBE_MULTIPLIERS[vibe];

  // Archetype type preference
  score *= ARCHETYPE_TYPE_MULTIPLIERS[archetype][type] || 1.0;

  // Tag bonuses
  tags.forEach(tag => {
    score *= TAG_MULTIPLIERS[tag] || 1.0;
  });

  // Duration modifier (if provided)
  if (duration) {
    score *= DURATION_MODIFIERS[duration];
  }

  return Math.round(score);
}
```

**Example:**
- Type: `meal-drink` (22 pts)
- Vibe: `FullMoon` (1.3x)
- Tags: `deep-talk` (1.3x), `one-on-one` (1.1x)
- Archetype: `HighPriestess` (meal-drink: 1.2x)
- **Total:** 22 √ó 1.3 √ó 1.3 √ó 1.1 √ó 1.2 = **49 points**

Compare to:
- Type: `text-call` (10 pts)
- Vibe: `WaxingCrescent` (1.0x)
- No tags
- **Total:** 10 points

**Deep meaningful meals matter way more than quick texts** - the system reflects reality.

## Open Questions to Resolve:

### 1. Acts of Care - Needs Refinement

**Current Issues:**
- Too broad? Covers both "brought soup when sick" and "listened during crisis"
- Vague icon (‚ù§Ô∏è‚Äçü©π vs ‚ù§Ô∏è vs ü§ù)
- Overlaps with other types (can happen during hangout, meal, etc.)

**Options:**
1. **Keep as standalone** for significant support moments
2. **Make it a tag** that can apply to any interaction (#ActsOfCare)
3. **Split into two**: "Showing Up" (active support) vs "Being Held" (receiving care)
4. **Rename to "Support"** for clarity

**My Recommendation:** Make it a **premium tag** that can enhance any interaction type:
- Had a meal where you supported them? ‚Üí `meal-drink` + `#ActsOfCare`
- Called during a crisis? ‚Üí `text-call` + `#ActsOfCare` + `#Vulnerable`
- This preserves the importance while reducing category bloat

### 2. Duration - Keep or Remove?

**Current:** Quick / Standard / Extended
**Question:** Is this still needed if we have tags like `#QuickCheckIn` or types like `text-call` (inherently quick)?

**Recommendation:** Keep but make it **auto-suggested**:
- `text-call` defaults to Quick
- `meal-drink` defaults to Standard
- User can override if needed

### 3. Voice Note - Own Type or Merged?

**Options:**
1. Merge into `text-call` (they're both quick async communication)
2. Keep separate (voice has more emotional bandwidth)

**Recommendation:** Keep separate with distinct icon üé§ - voice notes have personality that texts don't.
